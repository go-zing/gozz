# Tag

Manager struct field tag in templating mode.

# Usage

# Annotation

+zz:tag:[tag]:[format]

# Annotation Target

All TypeSpec object and struct fields in annotated type.

# Exact Arguments

# tag

Specify tag key, if there are multi key with same value, use , to separate them.

Example: +zz:tag:json,form,bson,sql:{{ snake .FieldName }}


Use tag begins with +, to append same key value.

Example:

package x

// +zz:tag:json:{{ snake .FieldName }}
type T struct {
	// +zz:tag:+json:,omitempty
	ObjectId string `json:"object_id,omitempty"`
}

# format

Specify generate tag value, could use template data contains FieldName and field doc Doc. Strings functions like snake / camel are also provided.

Example: +zz:tag:json,form,bson,sql:{{ snake .FieldName }}

# Example-01

Example Project (opens new window)

package tag01

import (
	"time"
)

//go:generate gozz run -p "tag" ./

// +zz:tag:json,bson:{{ snake .FieldName }}
type User struct {
	Id        string
	Name      string
	Address   string
	CreatedAt time.Time
	UpdatedAt time.Time
}

Use annotation to specify json bson in snake case FieldName

执行 gozz run -p "tag" ./

package tag01

import (
	"time"
)

//go:generate gozz run -p "tag" ./

// +zz:tag:json,bson:{{ snake .FieldName }}
type User struct {
	Id        string    `bson:"id" json:"id"`
	Name      string    `bson:"name" json:"name"`
	Address   string    `bson:"address" json:"address"`
	CreatedAt time.Time `bson:"created_at" json:"created_at"`
	UpdatedAt time.Time `bson:"updated_at" json:"updated_at"`
}

bson json tags were correctly generated.

New tags would follow letter order by key.
